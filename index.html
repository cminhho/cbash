CBASH CLI
GitHub Docs

Command-line toolkit for developers

A single CLI to automate daily tasks, manage Git and Docker, run dev services, and work with AWS and Kubernetes. Built for macOS and Linux. Install once, use from any terminal.

Install

Choose your preferred method. Requires Bash 4+ or Zsh, and Git. After install, restart your terminal and run cbash to see the help.

brew install cminhho/tap/cbash-cli
sh -c "$(curl -fsSL https://raw.githubusercontent.com/cminhho/cbash/master/tools/install.sh)"
sh -c "$(wget -qO- https://raw.githubusercontent.com/cminhho/cbash/master/tools/install.sh)"

Features

Plugins and aliases for setup, Git, Docker, cloud, and local workflows. Run cbash list-plugins to see everything available.

Setup

Check your dev environment, install tools via Homebrew (dev, cloud, IDE, apps), and scaffold a standard workspace layout.

Git

Manage branches, squash commits, pull across multiple repos, quick backup and push, open repo in browser. Includes short aliases (g, gst, gco, commit, etc.).

Dev & Docker

Start, stop, and inspect Docker-based dev services. Follow logs, exec into containers, view stats. Shortcuts for common tasks (dps, drm, prune, kill-all).

AWS

SSH into instances via SSM Session Manager, work with SQS (including LocalStack), and read SSM parameters from the CLI.

Kubernetes

List pods, follow logs, exec into pods, rollout restart deployments. Includes a quick kubectl cheat sheet for common commands.

Generators

Scaffold directories for troubleshooting or features, workspace layout, project structure, and doc templates from the CLI.

Docs & Cheatsheets

View and edit project docs and cheatsheets from the terminal. Optional community cheatsheets via cheat setup.

AI (Ollama)

Chat with local models via Ollama, list and pull models from the CLI. Handy for quick prompts without leaving the terminal.

macOS & Proxy

Lock screen, run speedtest, list listening ports and IPs, generate passwords, directory tree. Toggle proxy for env, npm, and git.

Usage

Commands are plugin-based. Run cbash or cbash help for the full reference; cbash <plugin> help for plugin-specific help.

cbash                  # Show main help
cbash <plugin>         # Run a plugin (e.g. git, dev, aws, k8s)
cbash list-plugins     # List all available plugins
